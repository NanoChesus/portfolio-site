Graze Invaders

This is a Galaga-like gallery shooter where you must defend your cattle from being abducted by alien invaders trying to starve America. Features include increasing difficulty, scorekeeping, and basic enemy AI. It was made in JavaScript using the Phaser API.

Challenges: Some of the challenges I encountered while making this game were relearning JavaScript, learning the Phaser framework, and getting used to object-oriented coding. During the two-week development of this game, I improved my object-oriented skills by learning how to store data in objects and create custom functions/methods to manipulate that data for gameplay features.

1: One of the major challenges was creating a flexible, data-driven enemy wave system to spawn enemy waves and make creating new levels more efficient. I achieved this by structuring wave data as objects, with keys describing delays, the number of enemies, and a "type" indicator. This allowed the `waveSpawner` method to know which enemies to spawn and which AI behavior to assign. This unique game logic made the game feel more dynamic and is some of the code I’m most proud of.

2: Another learning opportunity came when I had to ensure each alien had a unique abduction target. I learned how to add enemies into arrays, check data within those arrays, and slice items from them to avoid duplicating targets. A simple but necessary challenge that helped me understand how to work with data efficiently.

Code Highlights:

  • Modular JavaScript architecture using Phaser scenes for easy scaling
  • Collision logic and object pooling for performance
  • Custome Data-driven wave spawning system

Tini-Venture

Tini-Venture is a 3D environment where you're meant to experience a children's storybook in VR.

This project focuses on immersive worldbuilding and spatial storytelling in low-poly VR environments. Built using ShapesXR, Blender, and free assets from Sketchfab. My goal with this project was to build a beautiful and whimsical landscape with a simple story that guides you through the world.

Design Focus:

  • Point-and-click exploration using MetaControllers in VR
  • Sound-reactive environments and ambient narrative cues
  • Low-poly assets crafted and customized in Blender
  • Scenes designed in ShapesXR for spatial storytelling

Unpainted Ant: This is the low-poly ant that inhabits the Tini world. They come in red and black types. Since many ants were needed per scene and the environment already used a large amount of bandwidth, we needed something that looked good but used minimal geometry.

The ant has no texture because it made importing simpler. We imported one colorless model and then applied red or black color (depending on ant type) in the editor, instead of importing two separate models.

Guard Bee: This bee guards the hive, but the red ants prove too strong and defeat him. The model is painted and rigged so that it can be posed in a defeated state after the ants invade. I textured him to give a more intimidating look. There are only two guard bees, and I kept them low-poly to maintain visual consistency.

3D Character

This is a showcase of 'Chesus' a 3D character model in a running animation

This was a challenging yet rewarding project. I started by learning Blender, gradually building the model from the body to the hands and feet, and then sculpting the head and retopologizing it. I later brought the mesh into ZBrush for texturing. Finally, I rigged and animated the model, overcoming many technical issues along the way. This process helped me grow my 3D modeling and animation skills significantly and giving me a love for the process.
